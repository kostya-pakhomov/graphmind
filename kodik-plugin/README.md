# GraphMind — плагин памяти для Kodik

Подключает к вашему агенту постоянную общую память: граф знаний с причинными связями, семантический поиск, персист между сессиями. Один агент накопил знание о проекте — остальные агенты и люди им пользуются.

Плагин — тонкая обёртка над MCP-сервером **GraphMind** (Rust, в корне этого репозитория). Он даёт: подключение сервера, правило автопамяти, навык, суб-агента-куратора и хуки автоматизации.

## Что внутри

| Компонент | Файл | Назначение |
|---|---|---|
| MCP-сервер | `.kodik-plugin/.mcp.json` | Подключает `graphmind` по HTTP (`/mcp`) |
| Правило | `rules/graphmind-memory.md` | Протокол автопамяти (bootstrap → record → flush) |
| Навык | `skills/graphmind-memory/SKILL.md` | Вызываемая работа с памятью |
| Суб-агент | `agents/memory-curator.md` | Наводит порядок: recall, противоречия, пробелы |
| Хуки | `hooks/hooks.json` + `hooks/*.mjs` | Авто-напоминания записать память (опционально) |
| Шаблоны правил | `templates/kodik-rules/*.md` | Готовые файлы для `.kodik/rules/` вашего workspace (см. ниже) |

## Предварительно: поднять сервер GraphMind
Плагин подключается к локальному серверу GraphMind на `127.0.0.1:50052`. Соберите и запустите его по инструкции в корневом [`README.md`](../README.md):

```bash
cargo build --release --features mcp-server,rocksdb
GRAPHMIND_MCP_HTTP=1 ./target/release/graphmind-v2      # или: docker compose up -d
curl -s http://127.0.0.1:50052/health
```

Серверу нужны LLM- и embeddings-эндпоинты (RouterAI / локальная Ollama) — см. корневой README, раздел «Конфиг».

## Установка плагина

- **Из маркетплейса Kodik:** `Ctrl+Shift+X` → найдите «GraphMind» → Install.
- **Вручную:** скопируйте содержимое `kodik-plugin/` в плагины Kodik (или укажите путь при установке из GitHub).

После установки плагин включён по умолчанию. Компоненты получают неймспейс `graphmind:` (навык `graphmind:graphmind-memory`, суб-агент `graphmind:memory-curator`).

## Проверка

1. Новый чат агента — в контексте появляется секция «GraphMind — автопамять» (от хука `SessionStart`).
2. `bootstrap_memory({ cwd: "<путь проекта>", query: "тест", depth: 0 })` — вернёт имя workspace.
3. `propose_new_memory({ level: "L2", node_type: "atom", content: "Проверка" })` → `memory_query({ query: "Проверка" })` находит карточку.

## Транспорт

По умолчанию — Streamable HTTP (`http://127.0.0.1:50052/mcp`).

## Хуки (опционально)

Хуки в `hooks/hooks.json` адаптированы из проверенных Cursor-хуков и напоминают агенту сохранять память (`SessionStart` / `PostToolUse` / `Stop`). Требуют Node.js ≥18 в PATH. Основной надёжный механизм — правило `rules/graphmind-memory.md`; хуки — автоматизация поверх него.

## Шаблоны правил Kodik (`templates/kodik-rules/`)

Эта папка содержит готовые `.md`-файлы, которые можно положить в `<ваш-workspace>/.kodik/rules/`, чтобы подсказать Kodik-агенту ваши локальные конвенции. Агент видит их как постоянные правила для этого проекта.

Сейчас доступны:

| Файл | Назначение |
|---|---|
| `git-push-instructions.md` | Безопасный workflow `git commit` / `git push` (явные `git add`, без `-A`, осторожность с PowerShell) |
| `graphmind-memory.md` | Памятка по протоколу автопамяти, если хотите вынести её отдельным правилом (вместо `rules/graphmind-memory.md` внутри плагина) |

### Как установить

1. Скопируйте нужный файл в `<ваш-workspace>/.kodik/rules/`.
2. Откройте его и замените плейсхолдеры `<...>` на свои значения (remote, HEAD-коммит, имя workspace).
3. Перезапустите Kodik — правило подхватится автоматически.

Файлы **переносимы**: внутри них нет приватных данных автора плагина, только структура и инструкции.
