# GraphMind — память между сессиями (Kodik)

Правило для `.kodik/rules/`. MCP-сервер **graphmind** должен быть подключён (см. корневой `README.md`, раздел «Подключить агента»).

Справочник инструментов: [docs/TOOLS.md](../../docs/TOOLS.md)

## Старт сессии

```
bootstrap_memory({ cwd: "<PROJECT_ROOT>", query: "<суть задачи>", depth: 1 })
```

`<PROJECT_ROOT>` — абсолютный путь проекта (напр. `c:/my-app`). Один вызов = автоопределение workspace + семантический recall.

## В процессе

| Действие | Вызов |
|----------|------|
| Шаг завершён | `record_action({ summary })` |
| Факт этого репо | `propose_new_memory({ level: "L2", node_type: "atom", content })` |
| Кросс-проектное решение | `propose_new_memory({ level: "L2", node_type: "atom", scope: "global", content })` |

### Когда `scope: "global"` (кросс-проектная память)

- Общие паттерны: MCP, Docker, CI, конвенции языка/IDE, настройка самих hooks
- **Не** бизнес-логика конкретного приложения и **не** статусы задач одного репо

## Конец сессии

```
flush_session_memory({ summary: "итог: что сделано, ключевые решения" })
```

## Запреты

- Не полагаться на историю чата — только память через MCP
- Не дублировать `AGENTS.md` / `KODIK.md` — только решения и статусы, которых нет в файлах
