# Модель памяти

GraphMind хранит знания в виде графа с трёхуровневой иерархией.

## Уровни

| Уровень | Тип | Описание |
|---|---|---|
| **L0** | Кластер | Тематический кластер — объединяет несколько L1-доменов. Создаётся автоматически при консолидации. |
| **L1** | Домен | Предметная область. Связывает L2-атомы по смыслу. Создаётся автоматически. |
| **L2** | Атом | Атомарный факт, причина, следствие или правило. Основная единица памяти. Создаётся вручную через `propose_new_memory`. |

## Типы узлов L2

- **atom** — факт или наблюдение
- **cause** — причина
- **effect** — следствие
- **rule** — правило (IF/THEN)

## S0 — Кратковременная память

S0 — буфер текущей сессии. Действия агента записываются в S0, затем при `flush_session_memory` консолидируются в L2.

## Workspace'ы

Память изолирована между workspace'ами. Каждый workspace — отдельный раздел хранилища с собственным графом. Поддерживается переключение, архивирование и поиск пересечений между workspace'ами.

## Жизненный цикл

```
S0 (сессия) → flush → L2 (атом) → консолидация → L1 (домен) → L0 (кластер)
```

1. Агент работает, действия пишутся в S0
2. `flush_session_memory` — S0 сбрасывается в L2
3. `consolidate_workspace` — L2-атомы группируются в L1-домены и L0-кластеры

## Рёбра

Узлы связаны рёбрами с типизированными отношениями (cause→effect, parent→child и др.). Рёбра создаются через `link_nodes` и используются для обхода графа (`get_chain`, `suggest_related`).
