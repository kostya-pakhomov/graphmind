# CHANGELOG — GraphMind v2

## v2.0.0 — Initial public release

Постоянная общая память для ИИ-агентов на Rust. MCP-сервер с графом знаний, причинными связями и семантическим поиском.

### Возможности

- **50 MCP-инструментов**: слои памяти S0→L2→L1→L0 с консолидацией, воркспейсы, семантический поиск (эмбеддинги + косинусная близость), причинный слой на LLM (`find_contradictions`, `predict_risks`, `propose_causal_link`), фильтр доверия (`verify_input`), любопытство, ось планирования (`plan_*`).
- **Три транспорта на одном сервере**: stdio, SSE (`/sse`+`/message`), Streamable HTTP (`/mcp`).
- **Персист** между перезапусками (RocksDB) + семантический recall из накопленного.
- **Web UI** на `:7878`: переключатель воркспейсов, счётчики L2/L1/L0, дерево планов P0–P3, LLM-чат, панель настроек.
- **Сетевая безопасность**: по умолчанию принимает только локальные и частные адреса; публичные отклоняются, пока не выставлен `GRAPHMIND_ALLOW_EXTERNAL=true`.
- **TrustFirewall**: мягкий гейт на путях записи — подозрительное помечается как `Draft`, не отказывает.
- **MemoryOrchestrator**: авто-консолидация при ≥10 новых узлах или простое >1 ч.
- **Плагин для Kodik**: MCP + правило + навык + суб-агент-куратор + хуки.
- **Шаблоны для Cursor**: правила и хуки автоматизации памяти.

### Требования

- Rust (свежий stable), protoc, cmake, C++-компилятор, libclang.
- LLM endpoint (OpenAI-совместимый `/v1/chat/completions`) — для причинного слоя и консолидации.
- Embeddings endpoint (`/v1/embeddings`) — для семантического поискка.

Без LLM/эмбеддингов сервер поднимется, но поиск деградирует до ключевых слов, а причинный слой — до эвристики.

### Поддерживаемые агенты

Kodik, Cursor, Claude Code, Codex, OpenCode — любой MCP-совместимый клиент.
